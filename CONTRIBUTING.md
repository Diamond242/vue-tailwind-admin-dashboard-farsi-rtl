# Contributing

Thanks for taking the time to contribute. This project follows standard open-source practices and the same community expectations used across Tailwind-related projects (Contributor Covenant).

## Quick Start
1. Fork the repo and create a feature branch.
2. Install dependencies: `npm install`.
3. Run the app: `npm run dev`.
4. Before opening a PR:
   - `npm run lint`
   - `npm run type-check`

## Guidelines
- Keep all UI changes RTL-safe and responsive.
- Do not introduce hardcoded strings; use `src/locales/*`.
- Use logical utilities (`text-start`, `ms/me`, `ps/pe`) instead of left/right.
- Keep formatting consistent (Prettier + ESLint).

## Localization Rules
- Add new keys to **both** `src/locales/en` and `src/locales/fa`.
- Keep key names stable and descriptive.
- Avoid changing existing keys unless necessary (backward compatibility).

## Pull Requests
- Provide a clear summary and screenshots if UI changes are involved.
- Keep PRs focused and small when possible.

## Code of Conduct
By participating, you agree to follow `CODE_OF_CONDUCT.md`.
